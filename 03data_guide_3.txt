박싱과 언박싱
object 형식은 참조 형식이기 때문에 힙에 데이터를 할당합니다.
int 형식이나 double 형식은 값 형식이기 때문에 스택에 데이터를 할당하고요.

object 형식은 값 형식의 데이터를 힙에 할당하기 위한 "박싱(Boxing)"기능을 제공합니다.
object 형식에 값 형식의 데이터를 할당하려는 시도가 이루어지면 object 형식은 박싱을 수행해서
해당 데이터를 힙에 할당합니다.
박싱되어 있는 값을 꺼내 값 형식 변수에 저장하는 과정을 일컬어서 "언박싱(Unboxing)"이라고 합니다.
//일단 소괄호 (자료형)a를 박싱 언박싱으로 이해 object에 넣으면 박싱, 값 형식 변수에 (object)b하면 언박싱

데이터 형식 바꾸기
변수를 다른 데이터 형식의 변수에 옮겨 담는 것을 형식 변환(Type Conversion)이라고 합니다.

문자열을 숫자로, 숫자를 문자열로 변환   @중요(자주까먹음) 
Parse() 분석하다
int a = int.Parse("12345");
float b = float.Parse("123.45");

c#은 숫자 데이터 형식을 문자열로 바꾸는 방법도 마련해 놨습니다.
정수,실수 데이터 형식은 object로부터 물려받은 ToString() 메소드를 자신이 갖고 있는 수를 문자열로 변환하도록
재정의(Override)했습니다.
int c = 12345;
string d = c.ToString();

float e = 123.45;
string f = e.ToString();

코드
 int f = Convert.ToInt32(f);

상수(Constants)
const 데이터 변경 불가능

열거 형식 (Enumerator)
enum 열거_형식명 : 기반자료형 { 상수1, 상수2 ... }

enum은 변수가 아닌 새로운 형식
열거 형식의 각 요소는 기본적으로 컴파일러로부터 값을 자동으로 할당받지만,
프로그래머가 원하는 값을 직접 대입할 수도 있습니다.
예) enum 열거_형식명 { YES=10} //이후 1씩추가됨
